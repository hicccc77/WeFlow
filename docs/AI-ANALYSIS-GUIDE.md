# AI 分析模块操作与排障手册

本文档用于 AI 分析模块（群聊分析中的 AI识别 / 分析历史 / AI设置）联调、使用和故障定位。

## 1. 功能入口

- 入口页面：`群聊分析`
- 入口按钮：`AI识别`、`分析历史`、`AI设置`

## 2. 首次配置（必做）

进入 `AI设置` 页面，依次填写并保存：

- API Key
- Base URL（默认 `https://api.deepseek.com/v1`）
- 模型名称（默认 `deepseek-chat`）
- 预处理提示词
- 合并分析提示词

建议保留默认策略：

- 默认时间范围：30 天
- 默认内容类型：文本

## 3. 任务执行流程

在 `AI识别` 页面：

1. 选择开始/结束日期
2. 勾选内容类型（文本/图片/视频）
3. 点击 `开始 AI识别`

系统会自动执行：

- 时间切片（约 15 批）
- 串行批处理
- 批次失败重试（最多 3 次）
- 批次间 1 秒间隔
- 最终合并分析

## 4. 结果展示

- 结果以表格展示
- 列宽支持拖拽，宽度全局持久化
- 单元格点击自动复制
- 运行中显示计时器
- 成功后通知弹窗提示

## 5. 日志定位方法（推荐）

进入 `分析历史`：

- 可按任务过滤
- 可按日志级别过滤（INFO/WARN/ERROR）
- 支持关键词检索（message/step/code/taskId/payload）
- 支持分页与页大小调整
- 支持自动刷新间隔

关键操作：

- `导出日志`：导出当前筛选结果
- `仅导出错误链路`：只导出失败相关日志
- `复制诊断包`：复制任务 + 日志完整 JSON
- `复制问题摘要`：复制聚合摘要和高频步骤

## 6. 常见错误码与处理建议

- `TASK_FAILED`
  - 任务级失败，先检查 API 配置和网络
- `AI_OUTPUT_INVALID`
  - AI 返回格式异常，检查提示词是否强约束 JSON 输出
- `BATCH_SKIP`
  - 批次重试后失败，进入任务详情定位失败时间段并复跑
- `BATCH_RETRY`
  - 批次临时失败，关注 error 字段是否持续同类错误

## 7. 批次级故障复跑

在 `分析历史` 选中任务后：

- 查看 `失败批次定位`
- 点击 `复跑该失败批次`

该操作仅复跑失败时间段，适合快速复现与修复验证。

## 8. 联调验收清单

- AI设置保存后再次进入页面可回显
- AI识别任务可启动并写入历史
- 任务日志包含 code、durationMs、payload
- 非 JSON 结果在页面显式报错
- 导出日志与错误链路导出文件可打开
- 复制诊断包可直接粘贴给开发排障
